# プルリクエスト作成

GitHubにプルリクエストを作成してください。以下の手順で実行してください。

ベースブランチはユーザーが指定した場合はそれを使用し、指定がない場合はmainブランチを使用します。

1. 現在のブランチを確認：
   - 現在のブランチがベースブランチの場合はエラーとして中止
   - 未コミット・未ステージの変更は無視する
2. ベースブランチとの差分を確認（git diff <base-branch>...HEAD）
3. PR本文を作成：
   - git diff <base-branch>...HEAD でベースブランチとの差分全体を確認
   - 個別のコミット内容ではなく、ブランチ全体で何を変更したかに焦点を当てる
   - .github/workflows/pull_request_template.md の形式に従って日本語で説明を記載
   - レビュアーが読みやすいよう、簡潔で要点のみを記載する
   - 冗長な説明や詳細すぎる技術的説明は避ける
   - テンプレートの各項目を埋めるために必要な最小限の情報のみを記載
4. リモートとの同期状態を確認：
   - git rev-parse HEAD でローカルの最新コミットハッシュを取得
   - git ls-remote origin <current-branch> でリモートの最新コミットハッシュを取得
   - 両者が一致しない場合、またはリモートブランチが存在しない場合はエラーとして中止
5. プルリクエストを作成（gh pr create --base <base-branch>）
6. 作成されたPRのURLを表示

重要事項。

- ベースブランチはユーザーが指定した場合はそれを使用、未指定の場合はmainを使用
- PR本文はベースブランチとの差分全体に基づいて作成（個別のコミット単位ではない）
- PR本文は簡潔で読みやすく、レビュアーに必要な最小限の情報のみを記載
- push済みのブランチでの実行を前提とする
- 未コミット・未ステージの変更は無視する
- 現在のブランチがベースブランチと同じ場合は中止
- PRテンプレートの形式に従って日本語で説明を記載
- PR本文に「🤖 Generated with [Claude Code](https://claude.ai/code)」を追加しない
